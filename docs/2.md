- 这是因为一个**不带优化功能**的简易编译器，总能将一段**连续**的 C 代码翻译成对应的一段**连续**的机器指令

```c
int foo(int n) {
	int sum = 0;
	for (int i = 1; i <= n; i ++) {
		sum += i;
	}
	return sum;
}
```

- 用`-S`编译选项查看其汇编语言的代码，可以看出很明显的代码对应关系（mov、cmp、jmp、...），这就是编译器做的编译工作

- 当然也可以使用`objdump`对机器代码进行反汇编，以查看其汇编语言的代码
